# DEC-0032 適用記録 — 開発側運用負荷軽減をP0へ変更

- 状態: 決定済・実装開始
- 判断: 採用
- Build: 334
- 決定日: 2026-07-24

## 決定
初回セットアップウィザードその他のユーザー導入支援は不採用・後順位とし、開発側のGitHub更新、差分検査、再試行、監査作業の負荷軽減をP0とする。

## 優先順位
1. P0: 開発側更新作業の削減、配置信頼性、再試行、障害診断、監査自動化
2. P1: GitHub連携処理の性能・安定性・大規模差分検証
3. P2: 利用者向け初回導入ウィザード、チュートリアル、一般向けサポート

## Build334適用
- GitHub GET/HEAD読取APIの一時障害を最大3回自動再試行
- 429/502/503/504およびネットワーク一時障害を対象化
- Retry-Afterまたは指数バックオフを使用
- HTTP状態、GitHub Request ID、Rate Limit情報をエラーへ付加
- POST/PATCH書込みAPIは重複Commit・不明状態を避けるため自動再試行禁止
- 最後の配置実行だけを人間承認とする安全境界を維持

## 不採用・後順位
- 初回セットアップモード
- 利用者向け導入ウィザード
- 初心者向けチュートリアル
