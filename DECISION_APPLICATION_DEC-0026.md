# DEC-0026 — Rollback

- 状態: 決定済・実装済
- 判断: 採用
- 決定日: 2026-07-23
- 方針: Atomic Update実行前の現行Exportを、対象と同じ親ディレクトリの`.Export.rollback/<UTC時刻>-<token>`へ永続保存する。
- 復元: 指定バックアップを事前検証し、stagingへ複製・再検証後、`rename()`で現行Exportと原子的に切り替える。
- 復元前保護: 復元直前の現行Exportも新しいバックアップ世代として保存する。
- 排他制御: Atomic Updateと共通の更新ロックを使用し、並行処理を`UPDATE_LOCKED`で拒否する。
- 失敗時: バックアップ検証・複製・再検証に失敗した場合は現行Exportへ触れない。切替失敗時は直前の現行Exportを元のパスへ戻す。
- セキュリティ: バックアップ・対象・複製中のシンボリックリンクを拒否する。
