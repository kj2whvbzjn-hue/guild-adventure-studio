# GK Studio v1.4.0

## フェーズ4：クエスト・イベントデータベース

- クエストに失敗条件、前提クエスト、後続クエストを追加
- クエストに必要フラグ、設定フラグを追加
- フラグデータベースを追加
- フラグの分類、初期値、説明、参照元を管理
- 使用中フラグの誤削除を防止
- クエスト前提関係の循環検出を追加
- クエスト・イベントの未登録フラグ参照を検出
- 旧バージョンのJSONを自動補完して読み込み


## 提出物配布ルール（統一運用）
- すべての提出物（監査報告・検査結果・修正計画・成果物・ログ・チェックリスト）は必ずZIP形式で配布する。
- 単体ファイルのみで正式提出してはならない。
- ZIPには関連ログ、SHA-256一覧、マニフェスト等を含める。
- リリースチェックではZIP作成・CRC・展開確認・SHA確認を実施する。
