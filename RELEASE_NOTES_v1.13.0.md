# GK Studio v1.13.0 Release Notes

## 現在フェーズ
フェーズ9：編成スナップショット・比較検証

## 実装内容
- 味方・敵編成、最大Tick、ログ上限を名前付きで保存
- 保存編成の読込、上書き、削除
- 編成スナップショットJSON出力
- 2編成を同一シード範囲で一括比較
- 勝率、平均Tick、標準偏差、平均与ダメージ、平均回復、平均生存数を比較
- 比較結果CSV出力

## 修正
- Service Worker登録URLをBuild 230へ統一
- manifest、更新情報、キャッシュ番号をBuild 230へ統一
- 既存保存データとの互換性を維持


## 設計決定追記（2026-07-23）
### DEC-0001 採用
- GK Studioをゲーム制作支援ツール、ギルド冒険物語を実ゲームとして責務分離。

### DEC-0002 採用
- PHP受渡しは `Export/` を正式境界とする。
- 共通JSONメタデータ、manifest、SHA-256検証、バージョン分離方針を追加。
- Export雛形と共通JSON Schemaを同梱。


## 提出物配布ルール（統一運用）
- すべての提出物（監査報告・検査結果・修正計画・成果物・ログ・チェックリスト）は必ずZIP形式で配布する。
- 単体ファイルのみで正式提出してはならない。
- ZIPには関連ログ、SHA-256一覧、マニフェスト等を含める。
- リリースチェックではZIP作成・CRC・展開確認・SHA確認を実施する。
